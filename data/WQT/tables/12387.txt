# Exchange hybrid deployment coexistence  user and collaboration features before vs. after

| Capability | Before hybrid deployment | After hybrid deployment |
| --- | --- | --- |
| Cross-premises mailbox moves | Not available as an integrated experience | Supported mailbox moves between on-premises and Microsoft 365 are enabled |
| Unified address book experience | Separate GAL/visibility limitations across environments | Improved cross-premises address visibility via hybrid recipient objects |
| Cross-premises calendar sharing | Limited to within each environment | Calendar sharing can work between on-premises and Microsoft 365 users |
| Sharing of availability information | Only within the same environment | Availability sharing can span both environments |
| Organization relationship | Not configured | Organization relationship is configured to support hybrid sharing features |
| End-user experience consistency | Users perceive two separate systems | Users can interact across environments with fewer boundaries |
