    <tr><td>Step 5 — Distribution to Requester</td><td>Hub delivers the requested tracking information and an operation result (success/partial/failure) with reason codes.</td><td>Shared Data Hub → Requester</td><td>API response / EDI delivery; correlation request ID</td><td>Requester receives timeline; hub logs delivery and updates access metrics.</td></tr>
    <tr><td>Step 6 — Proactive Fan-out (Optional)</td><td>For subscribed parties, hub publishes updates when new events arrive or when thresholds are crossed.</td><td>Shared Data Hub → Subscribers (Carrier, Forwarder, Shipper, Consignee)</td><td>Webhook/event bus; push notification; periodic report</td><td>Subscriber state updated; hub records notification outcomes (delivered/failed).</td></tr>
    <tr><td>End Condition / Output</td><td>The request workflow closes when information is delivered (or denied) and the hub persists an audit trail.</td><td>Shared Data Hub; Requester; (Optional) Agency</td><td>Audit log; delivery receipt; denial record</td><td>Case closed with final status and retention timer for compliance.</td></tr>
