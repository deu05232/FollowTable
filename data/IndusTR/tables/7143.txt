    <tr><td>Prototype</td><td>BSTR SOF_SignXMLToSignatureValue(BSTR CertID, BSTR XmlToSign, BSTR ReferenceURI, LONG HashAlg, LONG SigAlg)</td></tr>
    <tr><td>Purpose</td><td>Sign XML content and return a standalone XMLDSIG signature block (or signature value) suitable for embedding by the caller.</td></tr>
    <tr><td>Description</td><td>This function produces signature output only. It does not verify existing signatures and does not parse signed XML. It is designed for applications that need to place the signature into a specific location or transport it separately.</td></tr>
    <tr><td rowspan="6">Parameters</td><td>CertID: Unique certificate identifier for selecting the signing key.</td></tr>
    <tr><td>XmlToSign: Original XML payload to be signed (string).</td></tr>
    <tr><td>ReferenceURI: Reference value used in <ds:Reference URI="...">; pass "" for whole-document signature.</td></tr>
    <tr><td>HashAlg: 1=SHA-256; 2=SHA-1; 3=SM3.</td></tr>
    <tr><td>SigAlg: 1=RSA; 2=SM2.</td></tr>
    <tr><td>ReturnFormat(implicit)</td><td>Return is an XML fragment containing <ds:Signature> with <ds:SignatureValue> and related nodes.</td></tr>
    <tr><td rowspan="3">Return Value</td><td>Success: XML signature fragment string (caller can insert into business XML).</td></tr>
    <tr><td>Failure: Empty string.</td></tr>
    <tr><td>Typical Use</td><td>Used when the application builds the final signed XML envelope itself but delegates signing to the SOF component.</td></tr>
    <tr><td>Signature Standards / Notes</td><td>For RSA-based XMLDSIG, signature construction aligns with RFC 3275; for SM2/SM3, the resulting signature encoding follows GB/T 25061-related conventions depending on implementation policy.</td></tr>
    <tr><td>Algorithm Constraints</td><td>When SigAlg=SM2, HashAlg must be SM3; when SigAlg=RSA, HashAlg may be SHA-1/SHA-256 (subject to policy).</td></tr>
